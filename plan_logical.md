# 数独100关 - 逻辑可推导 + 提示功能 实现方案

## 要解决的问题

### 问题1：推导死局
当前挖空方式是随机挖空，可能出现需要猜测（试数）才能解的单元格，玩家无法用纯逻辑推进。

**解决思路：** 生成谜题后，用**逻辑求解器**模拟玩家推理，只保留那些可以通过逻辑规则推导出的挖空。如果某个空无法通过逻辑规则填充，说明该空需要猜测，就不挖它。

### 问题2：提示功能
点击"提示"按钮，找出当前局面下可以通过逻辑规则推出的数字，并显示推导过程。

## 实现方案

### 一、逻辑求解规则（按优先级）

1. **唯一候选数（Naked Single）**：某个空格的候选数只剩1个
2. **隐式唯一（Hidden Single）**：在某行/列/宫中，某个数字只出现于1个候选位置
3. **区块排除（Locked Candidates / Pointing & Claiming）**：
   - 指向（Pointing）：某宫中的某个数字只出现在同一行/列，排除该行/列的其他宫
   - 宣称（Claiming）：某行/列中的某个数字只出现在同一宫，排除该宫的其他行/列
4. **数对排除（Naked Pair）**：某行/列/宫中两个格子只有相同的2个候选数，排除这两个数在该单元的其他位置
5. **隐式数对（Hidden Pair）**：某行/列/宫中，2个数字只出现在2个格子中
6. **三链数（Naked Triple）**：类似数对，但涉及3个格子和3个数字

> 对于"可逻辑推导"的要求，**规则1-3基本足够**覆盖所有日常难度。
> 规则4-6可选（用于专家/大师级别）。

### 二、生成流程

```
for each 难度:
  模板变换生成完整解
  按约束逐步挖空（而不是随机挖空）:
    while 还没挖到目标数量:
      1. 计算所有剩余空格的候选数
      2. 对每个空格，尝试用逻辑规则推导（模拟玩家）
      3. 过滤掉"推导不出"的空格
      4. 从"可推导"的空格中随机选一个挖掉
      5. 更新候选数
    end
end
```

关键：**每挖一个空，都确保它当前是"可逻辑推导"的**。这保证了整局游戏不存在死局。
